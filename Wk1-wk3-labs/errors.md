CRITICAL (or FATAL)
Use CRITICAL to log severe issues that can cause a script failure. CRITICAL is reserved for urgent errors that require immediate attention and intervention.

ERROR
Use ERROR to log error messages or any unexpected conditions that could impact your script's operation. Errors often indicate issues that you should address for the script to function correctly.

WARN
Use WARN to log information that might not be errors, but could lead to potential problems if not addressed. It serves as a warning about conditions that might need further investigation.

INFO
Use INFO to log general information about the script's progress or key events.

DEBUG
Use DEBUG to log detailed information for debugging purposes.

Log rotation


Log rotation is a preventive measure to manage log files efficiently, preventing them from becoming too large. It involves creating new files and archiving old ones, ensuring logs size is manageable.
